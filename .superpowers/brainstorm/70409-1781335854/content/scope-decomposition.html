Phase 5+ 四个子系统

🔔 主动感知

事件总线 → 规则引擎 → 匹配关注 → 推送通知

🔗 跨源推演

🧠 用户记忆

用户画像 + 个性化检索 + 长期记忆

"那个接口"→系统知道指哪个

🕸️ Agent网状互调

A2A消息协议 + 线索质量门禁 + Supervisor仲裁

Doc发现线索→主动调Code Agent

关键问题：如何分解？

A

按子系统独立交付

四个子系统各自走 brainstorming → design → plan → implement，按优先级串行或并行

风险最低，每个子系统可独立验收、独立上线

B

按依赖关系分层交付

先做"用户记忆"（基础数据支撑），再做"主动感知"（依赖用户画像），再做"跨源推演"（依赖网状互调），最后"网状互调"（架构改造）

稳妥，但依赖链较长

C

本次只设计"主动感知"原型

按照 PRD 第五章建议，先跑一个最高频场景原型（如 PRD 变更通知开发者），其他三个方向后续再议

快速验证价值，收集反馈后再扩展
