5 Agent 集成设计

图组合 · Token 预算 · 错误处理 · 状态隔离 · API 端点

集成方式：两级组合（非单一大图）

Supervisor 图调用 Worker 子图；API 层编排 Supervisor → Synthesis 两级流程。

API 端点: POST /api/v1/query
# Level 1: Supervisor 图（内部通过 Send API 调用 Worker 子图）
supervisor_result = await supervisor_graph.ainvoke(
    {"original_query": query, "conversation_history": history},
    config={"configurable": {"thread_id": session_id, "checkpoint_ns": "supervisor"}}
)
worker_outputs = supervisor_result["worker_outputs"]

# Level 2: Synthesis 图（仅当 cross-source 或有 worker 结果）
final_answer = await synthesis_graph.ainvoke(
    {"worker_outputs": worker_outputs, "original_query": query},
    config={"configurable": {"thread_id": session_id, "checkpoint_ns": "synthesis"}}
)

return {"answer": final_answer, "citations": ..., "trace": ...}

为什么两级而不是单一大图？

  • Supervisor 和 Synthesis 各自独立可测
  • 单源简单查询可以跳过 Synthesis（Worker 直接返回）
  • 调试时能清楚看到 Supervisor 产出什么、Synthesis 产出什么
  • checkpointer namespace 天然隔离

代价

  • API 层多一次图调用（~1ms 框架开销）
  • Synthesis 不能回溯 Supervisor 内部状态

Checkpointer Namespace 隔离

# 每个 Agent 独立 namespace，checkpoint 互不干扰
configs = {
    "supervisor": {"configurable": {"thread_id": sid, "checkpoint_ns": "supervisor"}},
    "doc_worker":  {"configurable": {"thread_id": sid, "checkpoint_ns": "doc"}},
    "code_worker": {"configurable": {"thread_id": sid, "checkpoint_ns": "code"}},
    "sql_worker":  {"configurable": {"thread_id": sid, "checkpoint_ns": "sql"}},
    "synthesis":   {"configurable": {"thread_id": sid, "checkpoint_ns": "synthesis"}},
}
# thread_id 相同 → 同一会话可回溯全部 Agent 的 checkpoint
# checkpoint_ns 不同 → Agent 间状态不互相覆盖
