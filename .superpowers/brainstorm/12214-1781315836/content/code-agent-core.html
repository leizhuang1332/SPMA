ripgrep 搜索 → 完备度判断 → AST 扩展循环

# 参照 Doc Agent 的 route→search→aggregate→assess→expand 模式
# Code Agent: route → search → assess → [expand → search] 循环

graph = StateGraph(CodeAgentState)

graph.add_node("route", route_node)          # Phase 0: 仓库路由
graph.add_node("search", search_node)        # Phase 1+2: 搜索词构造 + ripgrep
graph.add_node("assess", assess_node)        # 完备度判断 (L1/L2/L3)
graph.add_node("expand", expand_node)        # AST 调用图扩展

graph.set_entry_point("route")
graph.add_edge("route", "search")
graph.add_edge("search", "assess")
graph.add_conditional_edges("assess", should_continue,
    {"expand": "expand", "END": END})
graph.add_edge("expand", "search")

5 个节点的职责

节点 职责 LLM? 关键输出
route file_path_cache 查表 → 确定候选仓库（500→5） ❌ candidate_repos[]
search 搜索词构造 + ripgrep 分层执行（首次走 exact，后续走当前 fallback_layer） 按需(Haiku翻译) ripgrep_results[], fallback_layer
assess L1确定性→L2精确命中→L3 LLM兜底 仅L3(Haiku) assessment, convergence_reason
expand read_file + TreeSitter → calls/called_by/imports ❌ expanded_context[], 新搜索词
should_continue 条件边：assessment==converge OR round≥max → END ❌ "expand" | "END"

完备度判断——3 级递进（参照 Doc Agent）

级别 条件 结论 LLM?
L1 确定性 结果≥3 AND code_refs 精确命中 立即收敛 ❌
L2 调用深度 结果≥3 AND (call_depth≥2 OR 本轮无新增文件) 收敛 ❌
L3 LLM 兜底 以上都不满足 Haiku 判断充足/不足 ✅ Haiku

渐进式回退——4 层 fallback_layer

search_node 内部：按 fallback_layer 选择搜索策略
fallback_layer=0 (exact):  ripgrep -w -F "exact_term" --json
  ↓ 结果<3
fallback_layer=1 (stem):   ripgrep -w "stem_split_term" --json
  ↓ 结果<3
fallback_layer=2 (fuzzy):  ripgrep -i "fuzzy_pattern" --json
  ↓ 结果<3
fallback_layer=3 (llm):    Haiku 生成新搜索词 → ripgrep

收敛条件

def should_converge(state: CodeAgentState) -> bool:
    # 1. 精确命中 → 直接收敛
    if state.get("fallback_layer") == 0 and len(state["ripgrep_results"]) >= 3:
        return True
    # 2. 调用深度达到上限
    if state.get("call_depth", 0) >= 2:
        return True
    # 3. 本轮无新增文件
    if state.get("new_files_this_round", 0) == 0:
        return True
    return False  # 触发 LLM 完备度判断
