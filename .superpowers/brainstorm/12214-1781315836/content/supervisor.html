build_supervisor_graph() — LangGraph StateGraph + Send API
graph = StateGraph(SupervisorState)

# 节点
graph.add_node("classify_and_extract", classify_and_extract_node)  # 共享 Haiku 调用
graph.add_node("rewrite", rewrite_node)                      # 查询标准化+扩展+分解
graph.add_node("dispatch", dispatch_node)                    # 构造 WorkerDispatch[] → Send()
graph.add_node("doc_worker", doc_worker_wrapper)              # 调用 DocAgent 子图
graph.add_node("code_worker", code_worker_wrapper)            # 调用 CodeAgent 子图
graph.add_node("sql_worker", sql_worker_wrapper)              # 调用 SQLAgent 子图
graph.add_node("score", score_node)                          # 三维质量评分 (确定性)
graph.add_node("reschedule", reschedule_node)                # 注入 hints → 重派

# 边
graph.set_entry_point("classify_and_extract")
graph.add_edge("classify_and_extract", "rewrite")
graph.add_edge("rewrite", "dispatch")
# dispatch 返回 Send(doc/code/sql_worker) → 并行执行
# worker_outputs 通过 operator.add reducer 自动合并
graph.add_edge("dispatch", "score")
graph.add_conditional_edges("score", should_reschedule, {
    "reschedule": "reschedule",
    "converge": END,
})
graph.add_edge("reschedule", "dispatch")

分类+实体抽取（共享 Haiku 调用）

这是整个流程中最重要的 LLM 调用。一次 Haiku 调用，with_structured_output 同时产出两份结果：

classify_and_extract(query) → (ClassificationResult, ExtractedEntities):
    1. Haiku + with_structured_output → 同时输出两个 Pydantic 模型
    2. 规则兜底: 4条硬规则逐条检查——有遗漏则补刀
       - 含"多少/统计/报表" → 补 sql
       - 含 REQ-XXXXX → 补 doc
       - 含 .py/.java/.go/函数名 → 补 code
       - 极短模糊查询(<8字) → 三源全查
    3. 降级: Haiku 不可用 → Qwen3-8B → 纯规则

重调度策略

def should_reschedule(state) → "reschedule" | "converge":
    for worker_type, score in state.quality_scores.items():
        if score < 0.6 and state.reschedule_count < 2:
            # 从成功 Worker 提取 hints
            hints = extract_discovered_entities(state.successful_workers)
            return "reschedule"  # → reschedule_node → dispatch
    return "converge"  # → END, 流向 Synthesis Agent
