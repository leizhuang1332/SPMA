关键数据流

阶段 输入 输出 LLM
意图分类+实体抽取 原始 query ClassificationResult + ExtractedEntities Haiku (1次调用)
查询改写 query + entities WorkerDispatch[] (per source) Haiku (按需扩展/分解)
Worker 并行执行 WorkerDispatch[] WorkerOutput[] (Send API fan-in) 各 Agent 内部 LLM
质量评分 WorkerOutput[], 原始query QualityScore[] (确定性代码) 无
重调度(可选) 失败Worker + 成功实体 新WorkerDispatch 无/Haiku
融合生成 WorkerOutput[] (跨源) 最终答案 Sonnet

5 个 Agent 之间的契约

所有 Agent 通过 WorkerDispatch 和 WorkerOutput 通信，确保松耦合：

  • Supervisor：生产 WorkerDispatch[], 消费 WorkerOutput[]
  • Doc Agent：消费 WorkerDispatch(code="doc"), 生产 WorkerOutput
  • Code Agent：消费 WorkerDispatch(code="code"), 生产 WorkerOutput
  • SQL Agent：消费 WorkerDispatch(code="sql"), 生产 WorkerOutput
  • Synthesis Agent：消费 WorkerOutput[], 生产最终答案
